Instalación y Configuración de Servidor Arista (Untangle) como SUBSEDE para OpenVPN

Documento interno – Empresarial IT · 29 Nov 2025 · Redes Informáticas

Guía estandarizada para instalar Arista NG Firewall (Untangle) como SUBSEDE, integrarlo con la SEDE mediante OpenVPN site-to-site, y aplicar buenas prácticas de aislamiento y verificación.

1. Objetivo del documento

Este documento proporciona a los técnicos del Laboratorio Empresarial una guía paso a paso para:

  • Instalar y configurar un servidor Arista (Untangle) como SUBSEDE.
  • Integrarlo con la SEDE mediante OpenVPN site-to-site.
  • Garantizar aislamiento entre empresas, enrutamiento correcto y verificaciones.

2. Arquitectura del sistema Empresarial

La SEDE actúa como autoridad central (certificados, export de redes, control), y cada empresa dispone de su SUBSEDE independiente para mantener aislamiento.

3. Requisitos previos

3.1 Hardware / VM

  • CPU x86_64
  • RAM ≥ 4 GB (ideal 8 GB)
  • Disco ≥ 40–80 GB
  • 2 NIC mínimas: WAN / LAN

3.2 Recursos

  • ISO de Arista NG Firewall (Untangle)
  • USB booteable (si físico)
  • IPs WAN/LAN del cliente (o DHCP)
  • Datos del router: gateway, DNS, rango LAN
  • Acceso a la SEDE y permisos para crear Remote Clients tipo Network

4. Etapa 1 – Instalación y Setup Wizard Offline

4.1 Instalación base (ISO/USB)

  • No conectar el equipo a la red durante la instalación.
  • Arrancar desde ISO/USB.
  • Seleccionar disco e instalar (según procedimiento).

4.2 Acceso al asistente Offline

Tras reinicio, el sistema ofrece acceso al wizard. Alternativamente, conectar un PC en LAN y acceder a la consola web (IP por defecto del appliance).

4.3 Offline Setup Wizard – Paso a paso

Paso 1 – Configure the Server

  • Password admin (según política Empresarial)
  • Zona horaria: Atlantic/Canary
  • Tipo de servidor: Other
  • Admin Email (según política)

Modo de instalación

  • Router mode (recomendado en clientes) — no aplica en este caso.
  • Transparent bridge — caso SUBSEDE típico.

Paso 2 – Identify Network Cards

Asignar interfaces:

  • External (WAN)
  • Internal (LAN)

Paso 3 – Configure WAN

  • DHCP (habitual)
  • IP estática (IP, máscara, gateway, DNS)
  • PPPoE (si aplica)

Paso 4 – Configure LAN

Recomendación SUBSEDE: Transparent bridge y parámetros LAN según cliente.

Configuración alternativa en Dashboard

En CONFIG → Network se definen hostname, domain, puerto de administración HTTPS y reglas de acceso.

Paso 5 – Automatic Updates

  • LAB: desactivar updates automáticos.
  • Cliente: según política Empresarial.

4.4 Verificaciones tras el Wizard

Desde un PC en LAN:

ping 8.8.8.8
ping google.com

Acceso web (ejemplo):

https://192.168.0.240

4.5 Instalación de Apps (Reports y OpenVPN)

  1. Conectar a ETM Dashboard (si procede).
  2. Apps → Add → instalar Reports.
  3. Apps → Add → instalar OpenVPN.

5. Etapa 2 – Configuración de OpenVPN como SUBSEDE (site-to-site)

5.1 Preparación en SEDE

En la SEDE (normalmente por técnico senior), validar servidor OpenVPN habilitado y configurar Exported Networks y Remote Clients tipo Network.

5.2 Configurar SUBSEDE como cliente

En la SUBSEDE: Apps → OpenVPN → Client → subir el fichero descargado desde la SEDE.

5.3 Pruebas y verificación del túnel

  • SUBSEDE: ping hacia red SEDE (desde Troubleshooting)
  • SEDE: ping hacia red SUBSEDE

6. Modelo de acceso de usuarios remotos (Empresarial)

  • Solo la SEDE genera certificados.
  • Usuarios remotos se conectan a SEDE con su certificado.
  • La SEDE enruta hacia la SUBSEDE correspondiente.
  • SUBSEDE no expide certificados.

7. Aislamiento entre empresas / buenas prácticas

  • Firewall rules en SEDE para limitar acceso por empresa/SUBSEDE.
  • Policy Manager si se integra con AD/LDAP.
  • En SUBSEDE aceptar tráfico solo de pools/rangos autorizados.

8. Checklist final del técnico

ETAPA 1 – Instalación

  • ISO grabada y hardware verificado
  • Instalación base completada
  • Wizard offline completado
  • WAN operativa
  • LAN accesible
  • Reports instalado
  • OpenVPN instalado

ETAPA 2 – OpenVPN

  • SEDE: Server habilitado
  • SEDE: Exported Networks configuradas
  • SEDE: Remote Client creado
  • SUBSEDE: fichero cargado
  • SUBSEDE: estado Connected
  • Ping SEDE → SUBSEDE
  • Ping SUBSEDE → SEDE
