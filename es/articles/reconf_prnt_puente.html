Guía Técnica: Reconfiguración de Periféricos por Cambio de Subred

Rolando Luis Martín · 5 Mar 2026 · Soporte IT · Nivel: Avanzado

Guía técnica para resolver huérfanos de IP tras un cambio de Subred (router/ISP/Dominio).

Contenido
Figuras
Nota

Esta es una guía técnica estructurada para resolver conflictos de direccionamiento IP tras un cambio de infraestructura de red (router/ISP).

Objetivo

El objetivo es recuperar el acceso a periféricos con IPs estáticas sin necesidad de un reset de fábrica físico.

Contexto

1. Diagnóstico de la Situación

Al cambiar el router, la subred cambió de 192.168.1.0/24 a 172.26.3.0/24. Los equipos por DHCP se adaptaron automáticamente, pero las impresoras, al tener <>IP Fija (estática), quedaron "huérfanas" en una red que el nuevo router no reconoce.

Desarrollo

2. Identificación de Dispositivos (Advanced IP Scanner)

Para encontrar las impresoras sin adivinar, debemos escanear ambos rangos simultáneamente.

  • Acción: En la barra de direcciones de Advanced IP Scanner, define un rango amplio que cubra ambas subredes.

  • Rango a ingresar: 172.26.3.1-172.26.3.254, 192.168.1.1-192.168.1.254

Nota

Nota Visual: En el software, verás que puedes separar los rangos por comas. Esto obligará a la tarjeta de red a buscar respuestas en ambos segmentos.

3. El "Puente" de Red (Configuración IPv4)

Para que el PC de soporte "hable" con la impresora en la red vieja, necesitamos que la tarjeta de red del PC pertenezca a ambas redes a la vez.

  1. Ve a Propiedades de Ethernet -> Protocolo de Internet versión 4 (TCP/IPv4).

  2. Asigna una IP fija temporal en el rango nuevo (ej. 172.26.3.50).

  3. Haz clic en Opciones avanzadas... y en la sección Direcciones IP, pulsa "Agregar".

  4. Añade una IP del rango antiguo (ej. 192.168.1.50) con su respectiva máscara (255.255.255.0).

Nota

¿Por qué hacemos esto? Esto permite que tu navegador pueda cargar la interfaz web de la impresora aunque esta siga en la IP 192.168.1.X.

3. El "Puente" de Red (Configuración IPv4)

Con el "puente" activo, abre tu navegador y escribe la IP antigua de la impresora detectada en el escaneo.

  • Acción: Ve al apartado Network/Red -> IPv4.

  • Cambio: Cambia el modo de "Manual/Static" a las nuevas direcciones:

    • Impresora 1: 172.26.3.100

    • Impresora 2: 172.26.3.101

    • Máscara: 255.255.255.0

    • Puerta de enlace: 172.26.3.1 (La IP del nuevo router).

  • Importante: Aplica los cambios. Perderás conexión momentáneamente porque la impresora se ha movido a la nueva red.

5. Actualización de Puertos en Estaciones de Trabajo

No es necesario reinstalar drivers. Solo debemos apuntar el "túnel" de comunicación a la nueva dirección.

  1. En cada PC: Panel de Control -> Dispositivos e Impresoras.

  2. Clic derecho en la impresora -> Propiedades de impresora (no confundir con "Propiedades" a secas).

  3. Pestaña Puertos -> Configurar puerto... (o "Agregar puerto" -> Standard TCP/IP Port).

  4. Actualiza el nombre de la impresora o dirección IP a la nueva (172.26.3.100).

Configuración de Puerto Valor Sugerido
Nombre de puerto IP_172.26.3.100
Dirección IP 172.26.3.100
Protocolo RAW (Puerto 9100)

6. Verificación Final

  • Realiza una Página de prueba desde las propiedades de Windows.

  • Limpieza: Una vez terminado en todos los PCs, recuerda volver a poner la configuración IPv4 del PC de soporte en modo "Obtener una dirección IP automáticamente" (DHCP) para evitar conflictos futuros.

Tip

Puedes automatizar el proceso de configuración. El siguiente script de Powershell, realiza la mayoría de las acciones por ti

De hecho, para un Técnico IT, usar PowerShell es como tener un superpoder para ahorrar tiempo. En lugar de ir PC por PC haciendo clics en menús infinitos, puedes ejecutar un comando que reasigne el puerto de la impresora en segundos.

Aquí tienes el script y la explicación técnica de cómo funciona.

🚀 Script: "IT-Bridge & Printer Migrator"

Script Interactivo que hace todo el trabajo sucio por ti: configura el "puente" de red (añadiendo la IP vieja a la tarjeta), abre el panel de control para que el técnico vea los nombres de las impresoras y luego solicita los datos para reconfigurar el puerto.

El Código (PowerShell)

Copia este código en un archivo de texto, guárdalo como MigrarImpresora.ps1 y ejecútalo como Administrador.

# 1. --- CONFIGURACIÓN DEL PUENTE ---
Write-Host "=== PASO 1: CREANDO PUENTE DE RED (MULTIHOMING) ===" -ForegroundColor Yellow
$interfaz = (Get-NetAdapter | Where-Object Status -eq "Up")[0].Name

$ipNueva = Read-Host "IP ACTUAL del PC (ej. 172.26.3.50)"
$ipVieja = Read-Host "IP VIEJA para el puente (ej. 192.168.1.50)"

# Configuración de IPs
New-NetIPAddress -InterfaceAlias $interfaz -IPAddress $ipNueva -PrefixLength 24 -DefaultGateway "172.26.3.1"
New-NetIPAddress -InterfaceAlias $interfaz -IPAddress $ipVieja -PrefixLength 24
Write-Host "Red configurada. Acceso a ambas subredes activo." -ForegroundColor Green

# 2. --- IDENTIFICACIÓN (TU MEJORA) ---
Write-Host "`n=== PASO 2: SELECCIÓN DE IMPRESORA ===" -ForegroundColor Yellow
Start-Process "shell:::{A8A91A66-3A7D-4424-8D24-04E180695C7A}" # Tu comando pro
Write-Host "Copiando nombres de impresoras instaladas..." -ForegroundColor Cyan
Get-Printer | Select-Object Name | Format-Table

$nombreImpresora = Read-Host "Nombre EXACTO de la impresora"
$nuevaIPImpresora = Read-Host "NUEVA IP (ej. 172.26.3.100)"
$nuevoPuerto = "IP_$nuevaIPImpresora"

# 3. --- CAMBIO DE PUERTO ---
Add-PrinterPort -Name $nuevoPuerto -PrinterHostAddress $nuevaIPImpresora
Set-Printer -Name $nombreImpresora -PortName $nuevoPuerto
Write-Host "✅ Puerto migrado a $nuevaIPImpresora" -ForegroundColor Green

# 4. --- LIMPIEZA AUTOMÁTICA ---
Write-Host "`n=== PASO 4: FINALIZACIÓN Y LIMPIEZA ===" -ForegroundColor Yellow
$limpiar = Read-Host "¿Deseas restaurar el PC a DHCP automáticamente ahora? (S/N)"
if($limpiar -eq "S") {
    Set-NetIPInterface -InterfaceAlias $interfaz -Dhcp Enabled
    # Borramos las IPs estáticas temporales para evitar conflictos
    Get-NetIPAddress -InterfaceAlias $interfaz | Where-Object { $_.AddressFamily -eq "IPv4" } | Remove-NetIPAddress -Confirm:$false
    Write-Host "PC restaurado a DHCP. ¡Trabajo terminado!" -ForegroundColor Green
}

📘 Argumentación Técnica para el Técnico Junior

Es importante que el equipo entienda por qué usamos esto y no la interfaz gráfica:

  1. Consistencia: Evitas errores humanos al escribir la IP en máquinas distintas. Si el script funciona en una, funcionará en las restantes.

  2. Gestión de "Printer Spooler": PowerShell interactúa directamente con el servicio de cola de impresión, lo que suele ser más rápido y menos propenso a bloqueos que la ventana de "Propiedades de Impresora".

  3. Escalabilidad: Hoy son pocos ordenadores, pero si mañana son 50, este script te ahorra horas de trabajo manual.

¿Qué está pasando "bajo el capó" de este script?

  1. New-NetIPAddress: Es el comando de PowerShell que equivale a ir a las Propiedades Avanzadas de TCP/IPv4. Al ejecutarlo dos veces sobre la misma tarjeta (InterfaceAlias), Windows entiende que quieres multihoming (una tarjeta con varias IPs).

  2. shell:::{A8A91A66-3A7D-4424-8D24-04E180695C7A}: Este comando fuerza a Windows a abrir la Vista de Escritorio Clásica, que es donde podemos ver los nombres de puerto y propiedades avanzadas con total libertad.

  3. Try/Catch: Es una estructura de control de errores. Si el técnico escribe mal el nombre de la impresora, el script no "explota" con letras rojas feas, sino que da un mensaje controlado.

Advertencia

⚠️ Notas de Implementación

  • Nombre de la Impresora: El parámetro $nombreImpresora debe coincidir exactamente con el nombre que ves en Windows. Puedes listar todas tus impresoras instaladas con el comando Get-Printer | Select-Object Name para estar seguro.

  • Permisos: El script fallará si no se ejecuta como Administrador, ya que modificar puertos de impresión afecta a la configuración del sistema.

    • Al terminar el trabajo en los PCs, recuerda que el adaptador se ha quedado con IP fija. Para volver a DHCP y limpiar el "puente", el técnico debe ejecutar: 

    Set-NetIPInterface -InterfaceAlias "Ethernet" -Dhcp Enabled
    (O hacerlo manualmente en las propiedades de red).

Tip

💡 Tip de "Hacker" para el Técnico Junior:

Si alguna vez el CLSID se te olvida, puedes usar este comando en PowerShell para encontrar otros paneles ocultos: 

Get-ChildItem 'HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\ControlPanel\NameSpace'
