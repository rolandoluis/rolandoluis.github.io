Preparación de PCs en Laboratorio: Unión a Dominio vía VPN

Rolando Luis Martín · 8 Mar 2026 · Soporte IT · Nivel: Avanzado

Estrategia de pre-configuración técnica: maximiza la eficiencia uniendo equipos al dominio del cliente desde el laboratorio mediante un túnel VPN seguro.

Contenido
Figuras
Contexto operativo
Esta metodología permite realizar despliegues completos (políticas de grupo, unidades compartidas, M365, impresoras, redirección de carpetas) antes de la instalación física en las oficinas del cliente.

Objetivo

Establecer un estándar de procedimiento para la integración de equipos nuevos al dominio del cliente desde el laboratorio, utilizando conectividad VPN para asegurar la gestión remota total.

Requisitos previos

Antes de comenzar, asegúrese de tener a mano la siguiente información:

  • Nombre del equipo (PC) y nombre del dominio a unirse.
  • Dirección IP de la puerta de enlace (Gateway) del servidor VPN del cliente.
  • Credenciales válidas (usuario y contraseña) de un administrador del dominio.

Procedimiento

1. Configuración inicial del cliente VPN

Tras finalizar la instalación y habilitación del perfil base, proceda a instalar el software:

  • Descargue e instale la GUI del cliente desde OpenVPN Community.
  • Exporte el certificado de conexión desde el Servidor Sede e instálelo en el cliente.

2. Establecimiento del túnel y enrutamiento

Establezca la conexión VPN hacia el servidor en la red local del cliente. Una vez conectado:

  • Acceda a las conexiones de red ejecutando ⊞ + R y escribiendo ncpa.cpl.
  • Localice la interfaz Ethernet correspondiente al enlace VPN.
  • Edite sus propiedades e introduzca la puerta de enlace (Gateway) del servidor VPN del cliente para permitir el enrutamiento correcto hacia la red remota.

3. Unión al Dominio

No utilice el asistente de "Configuración avanzada". En su lugar:

  • Acceda a las propiedades del sistema (⊞ + R, sysdm.cpl).
  • Haga clic en el botón Cambiar.
  • Marque la opción Dominio e introduzca el nombre del mismo (ej: dominio.local\NombrePC).

También puede hacerlo mediante PowerShell 7, utilizando el siguiente comando: 

Add-Computer -DomainName "nombre.dominio" -Restart

4. Inicio de sesión post-unión

Reinicie el equipo como solicita el sistema. Una vez en la pantalla de inicio de sesión:

Atención
Para entrar, seleccione "Otro usuario". En el campo de usuario, debe introducir .\NombreUsuarioLocalAdmin. Es fundamental usar el punto y la barra invertida para forzar la autenticación local con el usuario con el que preparó la PC.

5. Gestión de usuarios del dominio

Una vez iniciada la sesión local, levante nuevamente la conexión VPN. Para integrar al usuario del dominio:

  • Ejecute ⊞ + R y escriba netplwiz.
  • Agregue el usuario del dominio y asígnele privilegios de administrador local para garantizar la compatibilidad con las aplicaciones.

6. Validación de la sesión del dominio

¡Muy Importante!
NO cierre la sesión, ya que perderá el túnel VPN. Utilice la función "Cambiar usuario" o "Switch user" desde el menú inicio para ir al muro de sesiones.

Desde el muro, seleccione "Otro usuario" e introduzca las credenciales del usuario del dominio. Al iniciar, se creará el perfil y podrá verificar la aplicación de las directivas (redirección de carpetas, 365, unidades, etc.).

Conclusión

Este flujo de trabajo transforma la entrega de equipos: al asegurar que las políticas y recursos están operativos desde el laboratorio, se garantiza una experiencia de usuario final impecable y se minimizan las intervenciones presenciales tras la entrega.
