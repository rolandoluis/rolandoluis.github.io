Activación del controlador StorNvme para garantizar el arranque tras clonar un sistema desde SATA a NVMe [cite: 1]

Rolando Luis Martín · 8 Mar 2026 · Soporte IT · Nivel: Avanzado

Aprende a preparar Windows para migraciones exitosas de SATA a NVMe, evitando el error INACCESSIBLE_BOOT_DEVICE mediante la configuración del controlador nativo StorNvme[cite: 7].

Nota
Esta guía es crucial cuando el sistema original se instaló en un disco SATA, ya que Windows no siempre tiene habilitado el controlador NVMe por defecto en esos casos[cite: 5].

Objetivo

Garantizar que Windows pueda arrancar correctamente en un nuevo dispositivo tras un proceso de clonación, utilizando comandos específicos para habilitar el controlador nativo de almacenamiento[cite: 5].

Introducción

Migrar de HDD o SATA SSD a unidades NVMe es cada vez más habitual en soporte IT por las mejoras en rendimiento[cite: 3]. Sin embargo, un problema crítico es que el sistema clonado a veces no arranca en el nuevo disco[cite: 4]. Esto sucede porque el controlador StorNvme puede estar configurado para iniciarse solo bajo demanda o estar desactivado si el sistema original no detectó unidades NVMe previamente[cite: 10, 11, 13].

Comprensión del problema

Cuando se intenta arrancar desde el NVMe clonado sin el controlador activo en la fase de BOOT, se producen errores como[cite: 15, 16]:

  • Pantalla azul (BSOD) INACCESSIBLE_BOOT_DEVICE[cite: 17].
  • Reboot infinito[cite: 18].
  • El BIOS detecta el disco, pero Windows no puede cargar el kernel[cite: 19].

¿Por qué "start= boot" es crucial?

El parámetro boot indica que el controlador debe cargarse en la fase más temprana del arranque[cite: 22]. Como la unidad NVMe contiene el sistema operativo, si el controlador no se carga desde el inicio, Windows es incapaz de leer el volumen de arranque[cite: 24, 25].

Guía práctica: Preparación y Clonación

Paso 1 – Habilitar el controlador NVMe

En el sistema original (SATA), abre una consola con privilegios de administrador y ejecuta[cite: 40, 41]:

sc config StorNvme start= boot

Para verificar que el cambio se aplicó correctamente, usa el comando[cite: 42, 43]:

sc qc StorNvme

Deberías ver en el resultado: START_TYPE : 0 BOOT_START[cite: 44].

Tip
Es vital realizar este paso antes de clonar para que la copia ya contenga la configuración necesaria para el primer arranque en el nuevo hardware[cite: 45, 46].

Paso 2 – Configuración de energía y clonado

Se recomienda desactivar el inicio rápido para evitar corrupción en la imagen[cite: 47, 48]:

powercfg -h off

Al clonar, asegúrate de incluir todas las particiones críticas: EFI, MSR y la partición del sistema (C:)[cite: 52, 53, 54].

Validación y Solución de problemas

Tras el primer arranque, puedes verificar el funcionamiento del controlador con[cite: 67, 68]:

Get-WindowsDriver -Online | findstr /i nvme
Advertencia: Si aparece BSOD
Si el sistema falla con INACCESSIBLE_BOOT_DEVICE, puede deberse a que el BIOS está en modo Legacy o la partición EFI está corrupta[cite: 75, 76]. En caso de corrupción, intenta reparar con:
bcdboot C:\Windows /f UEFI /l es-es[cite: 78].

Conclusión

La migración exitosa a NVMe depende de que el sistema reconozca el disco desde la etapa de BOOT[cite: 88, 91]. Configurar el controlador StorNvme antes de la clonación asegura migraciones limpias y profesionales, minimizando el tiempo de inactividad por errores de configuración[cite: 92].
