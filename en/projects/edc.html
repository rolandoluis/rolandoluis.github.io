EDC

Eurodat Deployment Console

Automation toolkit for provisioning Windows devices with a repeatable, phase-based workflow: offline installs, profile standardization, desktop/network policies and structured reporting.

← Back to Projects GitHub profile

Scope

Windows 10→11 migrations, provisioning flows, repeatable base user environment, technician-driven execution.

Core Idea

Deterministic steps with policies (STOP/CONTINUE), logs, state tracking and clean handoffs across phases.

Output

Consistent workstation setups + technical reports + reduced variance between technicians and machines.

Problem

Manual deployments are slow and inconsistent: different technicians produce different outcomes, updates break flows, offline environments require special handling, and troubleshooting lacks traceability.

Solution

EDC structures provisioning into phases (Fase 1/2/3) with explicit steps, predictable side-effects, and reusable modules for installation, configuration and auditing. Execution is logged and reportable.

Architecture

Modular PowerShell architecture (core utilities, dispatcher/menu, app install, desktop, update, express flows). State and logs stored per-run for traceability.

Workflow engine

Step results, policies, structured logging, and predictable error handling.

Offline-first

Curated repositories for apps and MSU packages to accelerate deployments.

Standardization

UserDefault → Tec → Usuario profile replication and desktop normalization.

Tech Stack

PowerShell · Windows 11 · Offline repositories · Deterministic workflows · Logging/Reporting
